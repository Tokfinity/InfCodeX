# v0.5.22 Feature Design: CLI-Based OAuth Providers

**Version**: v0.5.22
**Release Date**: 2026-03-08
**Status**: Design Review

---

## Overview

为 KodaX 添加两个新的 CLI-based Provider，采用 **CLI 命令包装模式**：
将 `codex exec` 和 `gemini -p` 作为子进程执行，解析其 JSON 输出，实现零维护、完全合规的 Provider 实现。

### 为什么选择 CLI 命令包装？

| 维度 | Token 缓存读取（原方案） | CLI 命令包装（新方案） |
|------|------------------------|----------------------|
| **合规性** | ⚠️ 有风险 | ✅ 完全合规 |
| **维护成本** | ❌ 高（需跟踪 token 格式变化） | ✅ 零维护 |
| **认证逻辑** | ❌ 需要自己实现 | ✅ CLI 负责 |
| **Token 刷新** | ❌ 需要自己实现 | ✅ CLI 自动处理 |
| **工具调用** | ❌ 需要自己实现 | ✅ CLI 内置 |
| **代码量** | ~1400 行 | ~500 行 |
| **性能** | ✅ 直接 HTTP | ⚠️ 进程启动开销 |

---

## Goals

1. **Gemini CLI Provider** - 通过 `gemini -p --output-format stream-json` 调用
2. **Codex CLI Provider** - 通过 `codex exec --json --full-auto` 调用
3. **流式输出** - 解析 JSON Lines 实现实时响应
4. **多轮对话** - 通过 session resume 支持上下文
5. **工具调用** - 使用 CLI 内置工具（文件操作、命令执行等）

## Key Components

### 1. CLI 执行器基类

```typescript
// providers/cli-executor.ts

import { spawn } from 'node:child_process';
import type { CLIExecutorConfig, CLIEvent, CLIExecutionOptions } from '../cli-events/types.js';

export abstract class CLIExecutor {
  protected config: CLIExecutorConfig;

  constructor(config: CLIExecutorConfig) {
    this.config = config;
  }

  /**
   * 检测 CLI 是否安装
   */
  async isInstalled(): Promise<boolean> {
    // 子类实现
  }

  /**
   * 获取 CLI 版本
   */
  async getVersion(): Promise<string | null> {
    // 子类实现
  }

  /**
   * 执行 CLI 并流式返回事件
   */
  async *execute(options: CLIExecutionOptions): AsyncGenerator<CLIEvent> {
    const args = this.buildArgs(options);
    const env = { ...process.env, ...this.config.env };

    const child = spawn(this.config.command, args, {
      cwd: this.config.cwd ?? process.cwd(),
      env,
      stdio: ['ignore', 'pipe', 'pipe'],
    });

    // 处理 abort 信号
    const abortHandler = () => child.kill('SIGTERM');
    options.signal?.addEventListener('abort', abortHandler);

    try {
      // 解析 JSON Lines 输出
      yield* this.parseOutputStream(child.stdout!, options.signal);
    } finally {
      options.signal?.removeEventListener('abort', abortHandler);
      child.kill();
    }
  }

  /**
   * 构建命令行参数
   */
  protected abstract buildArgs(options: CLIExecutionOptions): string[];

  /**
   * 解析输出流
   */
  protected async *parseOutputStream(
    stream: NodeJS.ReadableStream,
    signal?: AbortSignal
  ): AsyncGenerator<CLIEvent> {
    let buffer = '';

    for await (const chunk of stream) {
      if (signal?.aborted) break;

      buffer += chunk.toString();
      const lines = buffer.split('\n');
      buffer = lines.pop() ?? '';

      for (const line of lines) {
        const event = this.parseLine(line.trim());
        if (event) yield event;
      }
    }

    // 处理剩余 buffer
    if (buffer.trim()) {
      const event = this.parseLine(buffer.trim());
      if (event) yield event;
    }
  }

  /**
   * 解析单行 JSON
   */
  protected abstract parseLine(line: string): CLIEvent | null;
}
```

### 2. Codex CLI 执行器

```typescript
// cli-events/codex-parser.ts

import { CLIExecutor } from '../providers/cli-executor.js';
import type { CLIEvent, CLIExecutorConfig, CLIExecutionOptions } from './types.js';

/**
 * Codex CLI 原始事件类型
 * 参考: https://developers.openai.com/codex/noninteractive/
 */
interface CodexRawEvent {
  type: string;
  thread_id?: string;
  item?: {
    id: string;
    type: string;
    text?: string;
    command?: string;
    status?: string;
    name?: string;
    arguments?: string;
  };
  usage?: {
    input_tokens: number;
    cached_input_tokens: number;
    output_tokens: number;
  };
  message?: string;
  response?: unknown;
}

export class CodexCLIExecutor extends CLIExecutor {
  constructor(config?: Partial<CLIExecutorConfig>) {
    super({
      command: 'codex',
      baseArgs: ['exec', '--json', '--full-auto'],
      timeout: 300000,
      ...config,
    });
  }

  protected buildArgs(options: CLIExecutionOptions): string[] {
    const args = [...this.config.baseArgs];

    // 恢复会话
    if (options.sessionId) {
      args.push('resume', options.sessionId);
    }

    // 提示词
    args.push(options.prompt);

    return args;
  }

  protected parseLine(line: string): CLIEvent | null {
    if (!line.startsWith('{')) return null;

    try {
      const raw = JSON.parse(line) as CodexRawEvent;
      return this.convertEvent(raw);
    } catch {
      return null;
    }
  }

  private convertEvent(raw: CodexRawEvent): CLIEvent | null {
    const timestamp = Date.now();

    switch (raw.type) {
      case 'thread.started':
        return {
          type: 'session_start',
          timestamp,
          sessionId: raw.thread_id ?? '',
          model: 'codex',
          raw,
        };

      case 'item.completed':
        if (raw.item?.type === 'agent_message') {
          return {
            type: 'message',
            timestamp,
            role: 'assistant',
            content: raw.item.text ?? '',
            raw,
          };
        }
        // command_execution, file_change 等作为 tool_use
        if (raw.item?.type === 'command_execution') {
          return {
            type: 'tool_use',
            timestamp,
            toolId: raw.item.id,
            toolName: 'Bash',
            parameters: { command: raw.item.command },
            raw,
          };
        }
        return null;

      case 'turn.completed':
        return {
          type: 'complete',
          timestamp,
          status: 'success',
          usage: raw.usage ? {
            inputTokens: raw.usage.input_tokens,
            outputTokens: raw.usage.output_tokens,
            totalTokens: raw.usage.input_tokens + raw.usage.output_tokens,
          } : undefined,
          raw,
        };

      case 'error':
      case 'turn.failed':
        return {
          type: 'error',
          timestamp,
          errorType: raw.type,
          message: raw.message ?? 'Unknown error',
          raw,
        };

      default:
        return null;
    }
  }

  async isInstalled(): Promise<boolean> {
    try {
      const child = spawn('codex', ['--version']);
      return new Promise((resolve) => {
        child.on('close', (code) => resolve(code === 0));
        child.on('error', () => resolve(false));
      });
    } catch {
      return false;
    }
  }
}
```

### 3. Gemini CLI 执行器

```typescript
// cli-events/gemini-parser.ts

import { CLIExecutor } from '../providers/cli-executor.js';
import type { CLIEvent, CLIExecutorConfig, CLIExecutionOptions } from './types.js';

/**
 * Gemini CLI 原始事件类型
 * 参考: https://geminicli.com/docs/cli/headless/
 */
interface GeminiRawEvent {
  type: 'init' | 'message' | 'tool_use' | 'tool_result' | 'error' | 'result';
  timestamp?: string;
  session_id?: string;
  model?: string;
  role?: string;
  content?: string;
  delta?: boolean;
  tool_name?: string;
  tool_id?: string;
  parameters?: Record<string, unknown>;
  status?: string;
  output?: string;
  message?: string;
  stats?: {
    total_tokens?: number;
    input_tokens?: number;
    output_tokens?: number;
  };
}

export class GeminiCLIExecutor extends CLIExecutor {
  private model: string;

  constructor(config?: Partial<CLIExecutorConfig> & { model?: string }) {
    super({
      command: 'gemini',
      baseArgs: ['--output-format', 'stream-json', '--yolo'],
      timeout: 300000,
      ...config,
    });
    this.model = config?.model ?? 'gemini-2.5-pro';
  }

  protected buildArgs(options: CLIExecutionOptions): string[] {
    const args = ['-m', this.model, '-p', options.prompt];
    return [...args, ...this.config.baseArgs];
  }

  protected parseLine(line: string): CLIEvent | null {
    if (!line.startsWith('{')) return null;

    try {
      const raw = JSON.parse(line) as GeminiRawEvent;
      return this.convertEvent(raw);
    } catch {
      return null;
    }
  }

  private convertEvent(raw: GeminiRawEvent): CLIEvent | null {
    const timestamp = raw.timestamp ? Date.parse(raw.timestamp) : Date.now();

    switch (raw.type) {
      case 'init':
        return {
          type: 'session_start',
          timestamp,
          sessionId: raw.session_id ?? '',
          model: raw.model ?? this.model,
          raw,
        };

      case 'message':
        return {
          type: 'message',
          timestamp,
          role: raw.role as 'user' | 'assistant',
          content: raw.content ?? '',
          delta: raw.delta,
          raw,
        };

      case 'tool_use':
        return {
          type: 'tool_use',
          timestamp,
          toolId: raw.tool_id ?? '',
          toolName: raw.tool_name ?? '',
          parameters: raw.parameters ?? {},
          raw,
        };

      case 'tool_result':
        return {
          type: 'tool_result',
          timestamp,
          toolId: raw.tool_id ?? '',
          status: raw.status === 'success' ? 'success' : 'error',
          output: raw.output ?? '',
          raw,
        };

      case 'error':
        return {
          type: 'error',
          timestamp,
          errorType: 'error',
          message: raw.message ?? 'Unknown error',
          raw,
        };

      case 'result':
        return {
          type: 'complete',
          timestamp,
          status: raw.status === 'success' ? 'success' : 'failed',
          usage: raw.stats ? {
            inputTokens: raw.stats.input_tokens ?? 0,
            outputTokens: raw.stats.output_tokens ?? 0,
            totalTokens: raw.stats.total_tokens ?? 0,
          } : undefined,
          raw,
        };

      default:
        return null;
    }
  }
}
```

### 5. Provider 实现

```typescript
// providers/codex-cli.ts

import { KodaXBaseProvider } from './base.js';
import { CodexCLIExecutor } from '../cli-events/codex-parser.js';
import { CLISessionManager } from './cli-session-manager.js';
import type { KodaXMessage, KodaXStreamResult, KodaXProviderStreamOptions } from '../types.js';

export class KodaXCodexCliProvider extends KodaXBaseProvider {
  readonly name = 'codex-cli';
  readonly supportsThinking = false;

  private executor: CodexCLIExecutor;
  private sessionManager: CLISessionManager;

  constructor(sessionManager: CLISessionManager) {
    super();
    this.executor = new CodexCLIExecutor();
    this.sessionManager = sessionManager;
  }

  override async isConfigured(): Promise<boolean> {
    return this.executor.isInstalled();
  }

  async stream(
    messages: KodaXMessage[],
    _tools: KodaXToolDefinition[],
    _system: string,
    _thinking: boolean,
    streamOptions?: KodaXProviderStreamOptions,
    signal?: AbortSignal
  ): Promise<KodaXStreamResult> {
    // 检查 CLI 安装
    if (!await this.executor.isInstalled()) {
      throw new Error(
        'Codex CLI 未安装。\n\n' +
        '请运行:\n' +
        '  npm install -g @openai/codex\n' +
        '  codex login\n\n' +
        '登录后 KodaX 会自动使用。'
      );
    }

    // 获取或创建 session
    const kodaxSessionId = streamOptions?.sessionId ?? 'default';
    const existingSessionId = this.sessionManager.get(kodaxSessionId);

    // 构建提示词
    const prompt = this.buildPrompt(messages);

    // 执行 CLI
    const textBlocks: KodaXTextBlock[] = [];
    let currentText = '';
    let sessionId: string | undefined = existingSessionId;

    for await (const event of this.executor.execute({
      prompt,
      sessionId: existingSessionId,
      signal,
    })) {
      switch (event.type) {
        case 'session_start':
          sessionId = event.sessionId;
          this.sessionManager.set(kodaxSessionId, event.sessionId);
          break;

        case 'message':
          if (event.role === 'assistant' && event.content) {
            currentText += event.content;
            if (event.delta) {
              streamOptions?.onTextDelta?.(event.content);
            }
          }
          break;

        case 'tool_use':
          streamOptions?.onToolInputDelta?.(event.toolName, JSON.stringify(event.parameters));
          break;

        case 'error':
          throw new Error(`Codex CLI error: ${event.message}`);
      }
    }

    if (currentText) {
      textBlocks.push({ type: 'text', text: currentText });
    }

    return {
      textBlocks,
      toolBlocks: [],
      thinkingBlocks: [],
    };
  }

  private buildPrompt(messages: KodaXMessage[]): string {
    const parts: string[] = [];

    for (const msg of messages) {
      if (typeof msg.content === 'string') {
        parts.push(`${msg.role}: ${msg.content}`);
      } else {
        const text = msg.content
          .filter(b => b.type === 'text')
          .map(b => (b as { text: string }).text)
          .join('\n');
        if (text) {
          parts.push(`${msg.role}: ${text}`);
        }
      }
    }

    return parts.join('\n\n');
  }
}
```

## Three Key Problems Solved

### 1. 多轮对话（消息历史）

**问题**：CLI 非交互模式如何传递多轮对话？

**解决方案**：使用 Session Resume

```
首次调用:
  codex exec "分析这个项目"
  → 返回 session_id: "abc-123"

后续调用:
  codex exec resume abc-123 "继续优化"
  → CLI 自动恢复上下文
```

**实现**：
- `CLISessionManager` 维护 `KodaX Session → CLI Session` 映射
- 首次调用获取 session ID 并缓存
- 后续调用使用 `resume` 子命令

### 2. 工具调用（自定义工具）

**问题**：如何支持 KodaX 自定义工具？

**解决方案**：Phase 1 使用 CLI 内置工具

| Phase | 方案 | 说明 |
|-------|------|------|
| **Phase 1 (MVP)** | CLI 内置工具 | 使用 CLI 自带的文件操作、命令执行等 |
| Phase 2 | MCP Server | KodaX 作为 MCP Server 暴露自定义工具 |

**CLI 内置工具**：
- Bash (命令执行)
- Read/Write (文件操作)
- Web Search (网络搜索)
- Edit (代码编辑)

**这些已覆盖大部分代码助手需求，无需额外实现。**

### 3. 会话恢复

**问题**：如何支持会话恢复？

**解决方案**：Session ID 映射

```
┌─────────────────────────────────────────────────────────────┐
│                 CLISessionManager                           │
│  Map<KodaXSessionId, CLISessionId>                          │
│                                                             │
│  "kodax-user-123" → "codex-abc-456"                        │
│  "kodax-user-456" → "codex-xyz-789"                        │
└─────────────────────────────────────────────────────────────┘
```

**生命周期**：
1. KodaX Session 创建时，首次调用 CLI 获取 session ID
2. 后续调用使用 `resume` 恢复
3. KodaX Session 结束时，清理映射
