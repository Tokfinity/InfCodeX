## FEATURE_007: 主题系统完善

### 1. 概述

### 2. 影响范围

- `packages/repl/src/ui/contexts/` - 添加 ThemeContext

### 3. 技术方案

**ThemeContext**:
```typescript
interface ThemeContextValue {
  theme: Theme;
  setTheme: (name: string) => void;

```

**命令颜色迁移**:
```typescript
// 之前
console.log(chalk.cyan('消息'));

// 之后
```

- `/theme <name>` - 切换主题

3. **迁移 commands.ts 的 chalk 调用**
4. **迁移 project-commands.ts 的 chalk 调用**

## FEATURE_013: Command System 2.0

**Status**: Planned | **Priority**: High | **Created**: 2026-03-03

### 1. 概述

重构 KodaX 的 Command 系统，参考 pi-mono 的 Extension API 设计，实现：
- 动态命令注册机制
- 丰富的 UI 交互能力
- 统一的命令来源追踪
- 缺失命令补齐

### 2. 背景

#### 当前问题

1. **硬编码命令**: 所有命令定义在 `BUILTIN_COMMANDS` 数组中，无法动态添加
2. **无 UI 交互**: 命令无法提供 select/confirm/input 等交互对话框
3. **命令缺失**: 相比 pi-mono 缺失 12+ 实用命令
4. **无来源追踪**: 无法区分 builtin/extension/skill/prompt 来源

#### 对标系统: pi-mono

| 特性 | KodaX (现状) | pi-mono (目标) |
|------|-------------|----------------|
| 命令数量 | 17 | 18+ |
| 动态注册 | ❌ | ✅ `registerCommand()` |
| UI 交互 | ❌ | ✅ `ctx.ui.select/confirm/input` |
| 来源追踪 | ❌ | ✅ `source: extension/prompt/skill` |
| 参数补全 | ❌ | ✅ `getArgumentCompletions()` |

### 4. 技术方案

#### 4.1 CommandRegistry 核心类

```typescript
// packages/repl/src/commands/registry.ts

export type CommandSource = "builtin" | "extension" | "skill" | "prompt";

export interface CommandInfo {
  name: string;
  description?: string;
  source: CommandSource;
  location?: "user" | "project" | "path";
  path?: string;
}

export interface CommandDefinition {
  name: string;
  aliases?: string[];
  description: string;
  usage?: string;
  arguments?: CommandArgument[];
  source?: CommandSource;
  getArgumentCompletions?: (prefix: string) => Promise<Completion[] | null>;
  handler: CommandHandler;
}

export interface CommandContext {
  ui: UIContext;
  session: SessionContext;
  config: ConfigContext;
}

export interface UIContext {
  select(title: string, items: string[]): Promise<string | null>;
  confirm(title: string, message?: string): Promise<boolean>;
  input(title: string, defaultValue?: string): Promise<string | null>;
  notify(message: string, type: "info" | "warn" | "error"): void;
  setStatus(status: string): void;
}

class CommandRegistry {
  private commands = new Map<string, CommandDefinition>();

  register(def: CommandDefinition): void;
  unregister(name: string): boolean;
  get(name: string): CommandDefinition | undefined;
  getAll(): CommandInfo[];
  getBySource(source: CommandSource): CommandInfo[];
}
```

#### 4.2 ExtensionAPI 接口

```typescript
// packages/repl/src/commands/extension-api.ts

export interface ExtensionAPI {
  // 命令注册
  registerCommand(name: string, def: Omit<CommandDefinition, "name">): void;
  getCommands(): CommandInfo[];

  // 事件订阅
  on(event: "session_start" | "session_end" | "message_sent", handler: Function): void;
  off(event: string, handler: Function): void;

  // 配置访问
  getConfig(): Record<string, unknown>;
  setConfig(key: string, value: unknown): void;
}
```

#### 4.3 迁移现有命令

```typescript
  // ... 其他命令
```

```typescript
export interface CommandDefinition {
  // 基本信息
  name: string;                    // 命令名称 (不含 /)
  aliases?: string[];              // 别名
  description: string;             // 描述

  // 参数定义
  usage?: string;                  // 用法示例
  arguments?: CommandArgument[];   // 参数定义

  // 元数据
  source?: CommandSource;          // 来源

  // 交互能力
  getArgumentCompletions?: (prefix: string) => Promise<Completion[] | null>;

  // 处理器
  handler: (args: string, ctx: CommandContext) => Promise<void>;
}
```

```typescript
      await clipboard.write(lastMessage);

    }
  }
```

## FEATURE_015: Project Mode 2.0 - AI-Driven Development Workflow

**Status**: Planned | **Priority**: High | **Created**: 2026-03-07

### 1. 概述

将 Project 模式打造为 KodaX 的核心特色，提供完整的 AI-Driven 开发工作流，使其成为区别于其他 Coding Agent 的标志性功能。

### 2. 核心能力

#### 2.1 多轮头脑风暴 (Brainstorming Mode)

**命令**: `/project brainstorm "需求描述"`

**功能**:
- AI 自动提出探索性问题
- 质疑假设，挖掘隐藏需求
- 探索多种实现方案
- 生成决策树和方案对比表
- 记录完整讨论历史

**示例**:
```
/project brainstorm "需要实现一个用户权限系统"

AI: 让我们先明确几个关键问题：
1. 需要支持多少用户量级？
2. 权限粒度需要到什么级别？
3. 是否需要支持角色继承？
...
```

**实现**:
- 创建 `packages/repl/src/project/brainstorm.ts`
- 使用 LLM 生成探索性问题
- 存储讨论历史到 `.kodax/projects/{id}/brainstorm/`
- 支持从讨论直接生成 feature

#### 2.2 规划模式增强 (Enhanced Planning)

**命令**: `/project plan #n` 或 `/project plan "feature描述"`

**功能**:
- 自动生成三级结构计划:
  - **Phases**: 主要阶段（设计、开发、测试、部署）
  - **Milestones**: 关键里程碑
  - **Tasks**: 具体任务
- 估算每个任务的时间（基于历史数据）
- 评估复杂度（简单/中等/复杂）
- 生成依赖关系图（DAG）
- 甘特图可视化

**示例输出**:
```
## Phase 1: 设计 (Est: 2h)
- [ ] Task 1.1: 需求分析 (30min, 简单)
- [ ] Task 1.2: 技术选型 (1h, 中等)
  - 依赖: Task 1.1
- [ ] Task 1.3: 接口设计 (30min, 简单)
  - 依赖: Task 1.2

## Phase 2: 开发 (Est: 8h)
...
```

**实现**:
- 创建 `packages/repl/src/project/planner.ts`
- 集成历史数据分析（类似任务的实际耗时）
- 使用 Graphviz 生成依赖关系图
- 存储到 `.kodax/projects/{id}/plans/{feature_id}.json`

#### 2.3 长时间运行支持 (Long-Running Execution)

**命令**:
- `/project execute #n` - 启动长时间任务
- `/project checkpoint` - 保存当前进度
- `/project resume #n` - 恢复中断的任务

**功能**:
- 任务可跨多个会话持续执行
- 自动保存进度（每完成一个 task）
- 支持断点续传
- 显示整体进度条
- 估算剩余时间

**进度持久化**:
```json
{
  "feature_id": 15,
  "started_at": "2026-03-07T10:00:00Z",
  "current_phase": 2,
  "current_task": "2.3",
  "completed_tasks": ["1.1", "1.2", "1.3", "2.1", "2.2"],
  "total_tasks": 12,
  "estimated_remaining": "4h 30min",
  "checkpoints": [
    {"task": "1.3", "timestamp": "2026-03-07T11:30:00Z", "notes": "..."}
  ]
}
```

**实现**:
- 创建 `packages/repl/src/project/executor.ts`
- 实现任务调度器
- 集成到 session 恢复机制
- 添加进度显示 UI 组件

#### 2.4 质量闭环 (Quality Loop)

**命令**: `/project quality`

**工作流**:
```
┌─────────┐    ┌────────┐    ┌────────┐    ┌─────────┐
│ Coding  │───▶│ Review │───▶│  Test  │───▶│ Deploy  │
└─────────┘    └────────┘    └────────┘    └─────────┘
     │              │             │              │
     └──────────────┴─────────────┴──────────────┘
                    Quality Report
```

**功能**:
- **自动流转**: 每个阶段完成后自动进入下一阶段
- **报告生成**: 每阶段生成详细报告
  - Coding: 变更文件列表、代码统计
  - Review: 问题列表、修复建议
  - Test: 测试覆盖率、失败用例
  - Deploy: 部署状态、回滚方案
- **回退支持**: 可回退到任意阶段
- **质量指标**:
  - 代码质量分数
  - 测试覆盖率
  - Bug 密度
  - 技术债务

**示例**:
```
/project quality

## Quality Report for Feature #15

### Overall Score: 85/100

| Phase | Status | Score | Details |
|-------|--------|-------|---------|
| Coding | ✅ Completed | 90 | 12 files changed, +850/-120 lines |
| Review | ✅ Completed | 82 | 3 issues found, all fixed |
| Test | 🔄 In Progress | - | Coverage: 78% (target: 80%) |
| Deploy | ⏳ Pending | - | - |

### Issues
- ⚠️ Test coverage below target (78% < 80%)
  - Missing tests: utils.ts, parser.ts
  - Run: /project test --coverage
```

**实现**:
- 创建 `packages/repl/src/project/quality-loop.ts`
- 集成测试框架（vitest）
- 创建报告模板
- 实现回退机制

### 3. 智能特性

#### 3.1 智能分析

- **历史学习**: 基于过往项目自动调整估算
- **模式识别**: 识别常见任务模式，复用计划
- **建议引擎**: 主动提出优化建议

#### 3.2 风险预警

- **依赖风险**: 识别外部依赖的潜在问题
- **时间风险**: 预警可能延期的任务
- **技术风险**: 识别高复杂度或不确定性任务

#### 3.3 资源优化

- **并行任务**: 识别可并行执行的任务
- **优先级调整**: 动态调整任务优先级
- **阻塞处理**: 自动处理阻塞的任务

```typescript
async function* runKodaX(...) {
  while (!shouldStop) {
    // 1. 执行一轮 LLM 调用
    const response = await llm.stream(...);
    yield* processResponse(response);

    // 2. 检查是否有排队的用户输入
    if (events.hasPendingInputs?.()) {
      const pendingInputs = events.consumePendingInputs?.();
      if (pendingInputs && pendingInputs.length > 0) {
        // 3. 将排队输入作为下一轮的用户消息
        for (const input of pendingInputs) {
          messages.push({ role: 'user', content: input });
        }
        // 清空队列
        events.clearPendingInputs?.();
        // 继续循环（不增加 iteration 计数）
        continue;
      }
    }

    // 4. 正常的 iteration 逻辑
    iteration++;
    if (iteration >= maxIterations) break;
  }
}
```

### 3. 技术方案

#### 3.1 文件发现机制

```typescript
// packages/coding/src/context/agents-loader.ts

const CONTEXT_FILE_CANDIDATES = ["AGENTS.md", "CLAUDE.md"];

interface AgentsFile {
  path: string;
  content: string;
  scope: 'global' | 'project' | 'directory';
}

/**
 * 从指定目录加载上下文文件
 */
function loadContextFileFromDir(dir: string): AgentsFile | null {
  for (const filename of CONTEXT_FILE_CANDIDATES) {
    const filePath = join(dir, filename);
    if (existsSync(filePath)) {
      try {
        return {
          path: filePath,
          content: readFileSync(filePath, "utf-8"),
          scope: dir === getKodaxGlobalDir() ? 'global' : 'directory'
        };
      } catch (error) {
        console.warn(`Warning: Could not read ${filePath}: ${error}`);
      }
    }
  }
  return null;
}

/**
 * 加载项目级上下文文件
 * 优先级：全局 < 根目录 < ... < 当前目录 < .kodax/
 */
function loadProjectContextFiles(options: {
  cwd?: string;
  kodaxDir?: string;
}): AgentsFile[] {
  const resolvedCwd = options.cwd ?? process.cwd();
  const resolvedKodaxDir = options.kodaxDir ?? getKodaxGlobalDir();

  const contextFiles: AgentsFile[] = [];
  const seenPaths = new Set<string>();

  // 1. 全局配置 (~/.kodax/AGENTS.md)
  const globalContext = loadContextFileFromDir(resolvedKodaxDir);
  if (globalContext) {
    globalContext.scope = 'global';
    contextFiles.push(globalContext);
    seenPaths.add(globalContext.path);
  }

  // 2. 从根目录到当前目录遍历
  const ancestorFiles: AgentsFile[] = [];
  let currentDir = resolvedCwd;
  const root = resolve("/");

  while (true) {
    // 先检查 .kodax/AGENTS.md（更高优先级）
    const kodaxContextFile = join(currentDir, ".kodax", "AGENTS.md");
    if (existsSync(kodaxContextFile) && !seenPaths.has(kodaxContextFile)) {
      try {
        ancestorFiles.unshift({
          path: kodaxContextFile,
          content: readFileSync(kodaxContextFile, "utf-8"),
          scope: 'project'
        });
        seenPaths.add(kodaxContextFile);
      } catch (error) {
        console.warn(`Warning: Could not read ${kodaxContextFile}: ${error}`);
      }
    }

    // 再检查目录级 AGENTS.md
    const contextFile = loadContextFileFromDir(currentDir);
    if (contextFile && !seenPaths.has(contextFile.path)) {
      contextFile.scope = 'directory';
      ancestorFiles.unshift(contextFile);
      seenPaths.add(contextFile.path);
    }

    if (currentDir === root) break;

    const parentDir = resolve(currentDir, "..");
    if (parentDir === currentDir) break;
    currentDir = parentDir;
  }

  contextFiles.push(...ancestorFiles);
  return contextFiles;
}
```

```typescript
// packages/coding/src/prompts/builder.ts

export function buildSystemPrompt(options: {
  basePrompt: string;
  agentsFiles: AgentsFile[];
}): string {
  const { basePrompt, agentsFiles } = options;

  if (agentsFiles.length === 0) {
    return basePrompt;
  }

  // 构建上下文注入部分
  const contextSections = agentsFiles.map(file => {
    const scopeLabel = {
      'global': '全局规则',
      'project': '项目规则',
      'directory': '目录规则'
    }[file.scope];

    return `
## ${scopeLabel} (from ${file.path})

${file.content}
`;
  }).join('\n---\n');

  return `${basePrompt}

---

# Project Context

${contextSections}
`;
}
```
