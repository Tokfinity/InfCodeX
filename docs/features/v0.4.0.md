## FEATURE_005: v0.4.0 架构重构与模块解耦

**Status**: Planned
**Priority**: High
**Category**: Refactor

### 1. 需求概述

重大架构重构，创建三个高度解耦、可独立使用的模块：`@kodax/core`、`@kodax/cli`、`@kodax/repl`

**Goals**:
1. **@kodax/core**: 纯 AI 引擎，环境无关（Node.js、浏览器、边缘运行时）
2. **@kodax/cli**: 薄层 CLI 封装，带命令注册系统
3. **@kodax/repl**: 完整的交互式终端体验

**Key Changes**:
- 模块间标准化接口
- 事件驱动架构
- Provider 和 Tool 插件系统
- 独立的 npm 包

### 3. 影响范围

**当前问题**:
1. **遗留代码未清理**: `kodax_core.ts` 仍被引用，与 `core/` 功能重复
2. **CLI 职责过重**: `kodax_cli.ts` 包含 UI 层逻辑（Spinner、事件处理）
3. **模块边界模糊**: SessionStorage、事件处理归属不明确
4. **难以独立使用**: 无法将单个模块作为独立库使用

### 4. 技术方案

**目标架构**:
```
┌─────────────────────────────────────────────────────────────┐
│                      KodaX v0.4.0                           │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐   ┌──────────────┐   ┌──────────────┐     │
│  │  @kodax/core │ ← │  @kodax/cli  │ ← │  @kodax/repl │     │
│  │              │   │              │   │              │     │
│  │  核心引擎     │   │  CLI 入口    │   │  交互式终端   │     │
│  │  Provider    │   │  命令解析    │   │  Ink UI      │     │
│  │  Agent       │   │  参数处理    │   │  会话管理    │     │
│  │  Tools       │   │  配置管理    │   │  历史记录    │     │
│  └──────────────┘   └──────────────┘   └──────────────┘     │
│         ↑                  ↑                  ↑              │
│         └──────────────────┴──────────────────┘              │
│                          标准化接口                           │
└─────────────────────────────────────────────────────────────┘
```

**核心原则**:
| 模块 | 设计原则 | 职责边界 | 不应包含 |
|------|----------|----------|----------|
| `@kodax/core` | **极简、智能、无副作用** | 只负责 AI 交互 | ❌ fs/path, ❌ 配置文件, ❌ 全局状态 |
| `@kodax/cli` | **极简、智能、薄层** | 只负责命令解析和参数处理 | ❌ UI 组件, ❌ Spinner, ❌ 会话持久化 |
| `@kodax/repl` | **智能、清晰、高可靠** | 负责交互式体验和状态管理 | ❌ Provider 逻辑, ❌ Tool 执行 |

**模块能力矩阵**:

| 能力 | @kodax/core | @kodax/cli | @kodax/repl |
|------|:-----------:|:----------:|:-----------:|
| Provider 抽象层 | ✅ | - | - |
| Tool 执行引擎 | ✅ | - | - |
| Agent 主循环 | ✅ | - | - |
| 事件系统 | ✅ | - | - |
| Token 估算 | ✅ | - | - |
| System Prompt | ✅ | - | - |
| 参数解析 | - | ✅ | - |
| 命令注册 | - | ✅ | - |
| 配置读写 | - | ✅ | - |
| --init 逻辑 | - | ✅ | - |
| --team 逻辑 | - | ✅ | - |
| --auto-continue | - | ✅ | - |
| Ink UI | - | - | ✅ |
| Spinner | - | - | ✅ |
| FileSessionStorage | - | - | ✅ |
| 交互命令 | - | - | ✅ |
| 会话持久化 | - | - | ✅ |

```typescript
/**
 * KodaX 会话存储 - 文件系统实现
 */

import fs from 'fs/promises';
import fsSync from 'fs';
import path from 'path';
import os from 'os';
import { KodaXMessage, KodaXSessionStorage } from '../core/index.js';
import { getGitRoot } from '../cli/utils.js';

export const KODAX_SESSIONS_DIR = path.join(os.homedir(), '.kodax', 'sessions');

export class FileSessionStorage implements KodaXSessionStorage {
  async save(id: string, data: { messages: KodaXMessage[]; title: string; gitRoot: string }): Promise<void> {
    await fs.mkdir(KODAX_SESSIONS_DIR, { recursive: true });
    const meta = { _type: 'meta', title: data.title, id, gitRoot: data.gitRoot, createdAt: new Date().toISOString() };
    const lines = [JSON.stringify(meta), ...data.messages.map(m => JSON.stringify(m))];
    await fs.writeFile(path.join(KODAX_SESSIONS_DIR, `${id}.jsonl`), lines.join('\n'), 'utf-8');
  }

  async load(id: string): Promise<{ messages: KodaXMessage[]; title: string; gitRoot: string } | null> {
    const filePath = path.join(KODAX_SESSIONS_DIR, `${id}.jsonl`);
    if (!fsSync.existsSync(filePath)) return null;
    const lines = (await fs.readFile(filePath, 'utf-8')).trim().split('\n');
    const messages: KodaXMessage[] = [];
    let title = '', gitRoot = '';
    for (let i = 0; i < lines.length; i++) {
      const data = JSON.parse(lines[i]!);
      if (i === 0 && data._type === 'meta') { title = data.title ?? ''; gitRoot = data.gitRoot ?? ''; }
      else messages.push(data);
    }

    const currentGitRoot = await getGitRoot();
    if (currentGitRoot && gitRoot && currentGitRoot !== gitRoot) {
      console.log(chalk.yellow(`\n[Warning] Session project mismatch:`));
      console.log(`  Current:  ${currentGitRoot}`);
      console.log(`  Session:  ${gitRoot}`);
      console.log(`  Continuing anyway...\n`);
    }

    return { messages, title, gitRoot };
  }

  async list(gitRoot?: string): Promise<Array<{ id: string; title: string; msgCount: number }>> {
    await fs.mkdir(KODAX_SESSIONS_DIR, { recursive: true });
    const currentGitRoot = gitRoot ?? await getGitRoot();
    const files = (await fs.readdir(KODAX_SESSIONS_DIR)).filter(f => f.endsWith('.jsonl'));
    const sessions = [];
    for (const f of files) {
      try {
        const content = (await fs.readFile(path.join(KODAX_SESSIONS_DIR, f), 'utf-8')).trim();
        const firstLine = content.split('\n')[0];
        if (!firstLine) continue;
        const first = JSON.parse(firstLine);
        if (first._type === 'meta') {
          const sessionGitRoot = first.gitRoot ?? '';
          if (currentGitRoot && sessionGitRoot && currentGitRoot !== sessionGitRoot) continue;
          const lineCount = content.split('\n').length;
          sessions.push({ id: f.replace('.jsonl', ''), title: first.title ?? '', msgCount: lineCount - 1 });
        } else {
          const lineCount = content.split('\n').length;
          sessions.push({ id: f.replace('.jsonl', ''), title: '', msgCount: lineCount });
        }
      } catch { continue; }
    }
    return sessions.sort((a, b) => b.id.localeCompare(a.id)).slice(0, 10);
  }

  async delete(id: string): Promise<void> {
    const filePath = path.join(KODAX_SESSIONS_DIR, `${id}.jsonl`);
    if (fsSync.existsSync(filePath)) {
      await fs.unlink(filePath);
    }
  }

  async deleteAll(gitRoot?: string): Promise<void> {
    const currentGitRoot = gitRoot ?? await getGitRoot();
    const sessions = await this.list(currentGitRoot ?? undefined);
    for (const s of sessions) {
      await this.delete(s.id);
    }
  }
}
```

```typescript
  stop(): void;

    },
```

#### ISSUE_037: 两套键盘事件系统冲突

**优先级**: Medium | **状态**: 将在 v0.4.0 解决

**问题描述**:
- 项目存在两套键盘事件处理系统：
  1. `KeypressContext.tsx` - 优先级系统，支持多个处理器
  2. `InputPrompt.tsx` - 直接使用 Ink 的 `useInput`
- 两者无法同时使用，导致优先级系统无法用于 REPL

**解决方案** (Phase 5 实施时):
1. 迁移 `InputPrompt` 使用 `KeypressContext`
2. 注册输入处理器为 `KeypressHandlerPriority.Normal`
3. 注册全局快捷键为 `KeypressHandlerPriority.Critical`
4. 允许建议导航注册为 `KeypressHandlerPriority.High`

**代码修改**:

修改 `packages/repl/src/ui/components/InputPrompt.tsx`:
```typescript
// 旧代码 - 直接使用 useInput
useInput(handleInput, { isActive: focus });

// 新代码 - 使用 KeypressContext
import { useKeypressContext, KeypressHandlerPriority } from '../contexts/KeypressContext.js';

const { registerHandler, unregisterHandler } = useKeypressContext();

useEffect(() => {
  if (focus) {
    const handlerId = registerHandler(handleInput, KeypressHandlerPriority.Normal);
    return () => unregisterHandler(handlerId);
  }
}, [focus, handleInput, registerHandler, unregisterHandler]);
```

**验收标准**:
- [ ] InputPrompt 使用 KeypressContext 而非直接 useInput
- [ ] 全局快捷键（Ctrl+C, Ctrl+D）正常工作

#### ISSUE_040: REPL 显示严重问题 - 重复/乱序/占位符/命令不可见

**优先级**: High | **状态**: 将在 v0.4.0 解决

**问题描述**:
REPL 显示存在多个严重问题：
1. **消息重复显示**：同一条消息在终端中出现 2-3 次
2. **[Complex content] 占位符**：LLM 回复显示为占位符而非实际文本
3. **命令输出不可见**：`/help`、`/model` 等命令的输出不显示
4. **显示顺序混乱**：Banner、用户输入、LLM 回复的顺序完全错乱

**根因分析**:
1. **双重输出通道**：console.log + MessageList 两个独立的输出系统
2. **patchConsole 限制**：Ink 的 patchConsole 无法精确控制输出位置
3. **extractTextContent 不完整**：只处理 text 块，不处理 tool_use/tool_result 等

**解决方案** (Phase 5 实施时):

参考 Gemini CLI 实现 ConsolePatcher 模式，将所有输出统一为 HistoryItem。

**Step 1: 创建 ConsolePatcher 类**

创建 `packages/repl/src/utils/ConsolePatcher.ts`:

```typescript
/**
 * ConsolePatcher - 捕获 console.log 并转换为 HistoryItem
 *
 * 参考 Gemini CLI 的实现模式，将 console 输出统一到 MessageList 渲染。
 */

import type { HistoryItem } from '../ui/types.js';

type ConsoleLevel = 'log' | 'warn' | 'error' | 'info';

export interface ConsolePatcherOptions {
  onNewHistoryItem: (item: HistoryItem) => void;
}

export class ConsolePatcher {
  private onNewHistoryItem: (item: HistoryItem) => void;
  private originalConsole: {
    log: typeof console.log;
    warn: typeof console.warn;
    error: typeof console.error;
    info: typeof console.info;
  };
  private installed = false;

  constructor(options: ConsolePatcherOptions) {
    this.onNewHistoryItem = options.onNewHistoryItem;
    this.originalConsole = {
      log: console.log.bind(console),
      warn: console.warn.bind(console),
      error: console.error.bind(console),
      info: console.info.bind(console),
    };
  }

  /**
   * 安装 console 拦截器
   */
  install(): void {
    if (this.installed) return;

    console.log = this.createPatchedMethod('log');
    console.warn = this.createPatchedMethod('warn');
    console.error = this.createPatchedMethod('error');
    console.info = this.createPatchedMethod('info');

    this.installed = true;
  }

  /**
   * 卸载 console 拦截器，恢复原始 console
   */
  uninstall(): void {
    if (!this.installed) return;

    console.log = this.originalConsole.log;
    console.warn = this.originalConsole.warn;
    console.error = this.originalConsole.error;
    console.info = this.originalConsole.info;

    this.installed = false;
  }

  /**
   * 创建打补丁的 console 方法
   */
  private createPatchedMethod(level: ConsoleLevel): (...args: unknown[]) => void {
    return (...args: unknown[]) => {
      // 仍然输出到原始 console（用于调试日志等）
      // 但在 Ink 渲染时，这些会被过滤掉
      this.originalConsole[level](...args);

      // 转换为 HistoryItem
      const item = this.createHistoryItem(level, args);
      if (item) {
        this.onNewHistoryItem(item);
      }
    };
  }

  /**
   * 将 console 参数转换为 HistoryItem
   */
  private createHistoryItem(level: ConsoleLevel, args: unknown[]): HistoryItem | null {
    if (args.length === 0) return null;

    const text = this.formatArgs(args);
    if (!text.trim()) return null;

    const baseItem = {
      id: `console-${Date.now()}-${Math.random().toString(36).slice(2, 9)}`,
      timestamp: Date.now(),
    };

    // 根据级别选择 HistoryItem 类型
    switch (level) {
      case 'error':
        return { ...baseItem, type: 'error', text };
      case 'warn':
        return {
          ...baseItem,
          type: 'info',
          text: `⚠️ ${text}`,
          icon: '⚠️',
        } as HistoryItem;
      case 'info':
        return { ...baseItem, type: 'info', text };
      case 'log':
      default:
        return { ...baseItem, type: 'info', text };
    }
  }

  /**
   * 格式化 console 参数为字符串
   */
  private formatArgs(args: unknown[]): string {
    return args
      .map((arg) => {
        if (typeof arg === 'string') return arg;
        if (arg instanceof Error) return arg.stack || arg.message;
        try {
          return JSON.stringify(arg, null, 2);
        } catch {
          return String(arg);
        }
      })
      .join(' ');
  }
}
```

```typescript
/**
 * 从消息内容中提取文本
 * 支持所有 Anthropic 内容块类型
 */
export function extractTextContent(content: string | unknown[]): string {
  if (typeof content === 'string') {
    return content;
  }

  if (Array.isArray(content)) {
    const parts: string[] = [];

    for (const block of content) {
      if (!block || typeof block !== 'object') continue;

      const typedBlock = block as Record<string, unknown>;
      const blockType = typedBlock.type as string | undefined;

      switch (blockType) {
        case 'text':
          // 纯文本块
          if ('text' in typedBlock) {
            parts.push(String(typedBlock.text));
          }
          break;

        case 'tool_use':
          // 工具调用请求
          parts.push(formatToolUse(typedBlock));
          break;

        case 'tool_result':
          // 工具执行结果
          parts.push(formatToolResult(typedBlock));
          break;

        case 'thinking':
          // 思考内容
          if ('thinking' in typedBlock) {
            parts.push(`[Thinking] ${String(typedBlock.thinking)}`);
          }
          break;

        case 'image':
          // 图像内容
          parts.push('[Image]');
          break;

        case 'redacted_thinking':
          // 已编辑的思考内容
          parts.push('[Redacted thinking]');
          break;

        default:
          // 未知类型，尝试提取有用信息
          if ('text' in typedBlock) {
            parts.push(String(typedBlock.text));
          } else {
            parts.push(`[${blockType || 'Unknown block'}]`);
          }
      }
    }

    if (parts.length > 0) {
      return parts.filter(Boolean).join('\n');
    }
  }

  return '[No content]';
}

/**
 * 格式化 tool_use 块
 */
function formatToolUse(block: Record<string, unknown>): string {
  const name = block.name as string | undefined;
  const input = block.input as Record<string, unknown> | undefined;

  let inputPreview = '';
  if (input) {
    try {
      const json = JSON.stringify(input);
      inputPreview = json.length > 100 ? `${json.slice(0, 100)}...` : json;
    } catch {
      inputPreview = '[Complex input]';
    }
  }

  return `[Tool: ${name ?? 'unknown'}] ${inputPreview}`;
}

/**
 * 格式化 tool_result 块
 */
function formatToolResult(block: Record<string, unknown>): string {
  const content = block.content as string | unknown[];

  if (typeof content === 'string') {
    const preview = content.length > 200 ? `${content.slice(0, 200)}...` : content;
    return `[Result] ${preview}`;
  }

  if (Array.isArray(content)) {
    // 递归处理嵌套内容
    return `[Result] ${extractTextContent(content)}`;
  }

  return '[Result: empty]';
}
```

#### 方式 2: 手动脚本

创建 `scripts/sync-versions.js`：
```javascript
#!/usr/bin/env node
import fs from 'fs';
import path from 'path';

const rootPkg = JSON.parse(fs.readFileSync('package.json'));
const version = rootPkg.version;

const packages = ['core', 'cli', 'repl'];
for (const pkg of packages) {
  const pkgPath = path.join('packages', pkg, 'package.json');
  const pkgJson = JSON.parse(fs.readFileSync(pkgPath));
  pkgJson.version = version;

  // 更新内部依赖版本
  if (pkgJson.dependencies) {
    for (const dep of Object.keys(pkgJson.dependencies)) {
      if (dep.startsWith('@kodax/')) {
        pkgJson.dependencies[dep] = version;
      }
    }
  }

  fs.writeFileSync(pkgPath, JSON.stringify(pkgJson, null, 2) + '\n');
  console.log(`✅ Synced ${pkg} to version ${version}`);
}
```


#### 版本同步检查脚本

创建 `scripts/check-versions.js`：
```javascript
#!/usr/bin/env node
import fs from 'fs';

const rootVersion = JSON.parse(fs.readFileSync('package.json')).version;
const packages = ['core', 'cli', 'repl'];

let hasError = false;
for (const pkg of packages) {
  const pkgPath = `packages/${pkg}/package.json`;
  const pkgVersion = JSON.parse(fs.readFileSync(pkgPath)).version;
  if (pkgVersion !== rootVersion) {
    console.error(`❌ ${pkg}: ${pkgVersion} !== ${rootVersion}`);
    hasError = true;
  } else {
    console.log(`✅ ${pkg}: ${pkgVersion}`);
  }
}

process.exit(hasError ? 1 : 0);
```

## 常见问题 (FAQ)

### Q1: 为什么要用 monorepo 而不是多仓库？

| 方面 | Monorepo | 多仓库 |
|------|----------|--------|
| 代码共享 | ✅ 容易 | ❌ 需要 npm 发布 |
| 版本同步 | ✅ 统一 | ❌ 各自独立 |
| 重构 | ✅ 一次完成 | ❌ 需要协调多个仓库 |
| CI/CD | ✅ 统一配置 | ❌ 各自配置 |
| 发布 | ⚠️ 需要顺序发布 | ✅ 独立发布 |

### Q2: workspaces 和 lerna/changesets 有什么区别？

| 工具 | 功能 | 是否需要 |
|------|------|----------|
| npm workspaces | 基础 monorepo 支持 | ✅ 必需（已内置） |
| lerna | 版本管理、发布自动化 | ⚠️ 可选 |
| changesets | 变更日志、版本管理 | ⚠️ 可选 |

**KodaX 选择**：仅使用 npm workspaces，保持简单。如果未来需要更复杂的发布流程，可以考虑添加 changesets。

### Q3: 如何在本地测试未发布的包？

```bash
# 方式 1：使用 workspaces（推荐）
# 在 KodaX 根目录
npm install
npm link -g

# 方式 2：在其他项目中测试
cd /path/to/other-project
npm link @kodax/core  # 链接到本地 @kodax/core
```

### Q4: 发布顺序错误会怎样？

如果先发布 `@kodax/cli` 但 `@kodax/core` 还没发布，npm 会报错：
```
npm ERR! 404 Not Found - GET https://registry.npmjs.org/@kodax/core
```

**解决方法**：严格按照 `core → cli → repl` 顺序发布。

### Q5: 如何回滚到重构前的状态？

```bash
# 找到 Phase 0 之前的 commit
git log --oneline | grep -i "phase0"

# 回滚到该 commit
git reset --hard <commit-hash>

# 强制推送（谨慎使用）
git push origin main --force
```

### Q6: TypeScript Project References 是必需的吗？

**不是必需的**，但强烈推荐：
- 不使用：每次构建都会重新编译所有文件
- 使用：增量构建，只编译修改过的包

如果不想使用，可以跳过 Step 2.12，使用各 package 独立的 `tsc` 命令。
