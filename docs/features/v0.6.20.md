## FEATURE_041: Tool Output Guardrails and Context Overflow Protection

**Status**: Completed
**Priority**: High
**Category**: Enhancement

### Overview

This feature moves the protection boundary forward from compaction to tool output handling so a single oversized tool result can no longer blow up the conversation history before compaction gets a chance to help.

### Implemented

1. Shared tool-output truncation utilities
   - Added `packages/coding/src/tools/truncate.ts`
   - Added shared head/tail truncation, byte/line limits, line shortening, and spill-file helpers

2. Global tool-result guardrail before history writes
   - Added `packages/coding/src/tools/tool-result-policy.ts`
   - Applied the guardrail in `packages/coding/src/agent.ts` before tool results are emitted to the UI or written back into `messages`

3. Safer `read`
   - Reworked `packages/coding/src/tools/read.ts`
   - Added binary-file rejection
   - Added bounded streaming reads instead of whole-file `readFile()`
   - Added large-file preflight guidance and explicit `offset/limit` continuation hints
   - Added long-line shortening

4. Safer `bash`
   - Reworked `packages/coding/src/tools/bash.ts`
   - Replaced unbounded stdout/stderr accumulation with bounded tail capture
   - Added tail-oriented truncation for large command output
   - Included stderr in timeout previews

5. Safer `write` / `edit` / `grep`
   - `write` and `edit` now summarize oversized diffs and spill the full diff when needed
   - `grep` now shortens overly long match lines and truncates oversized result sets with follow-up hints

6. Tool contract and prompt guidance updates
   - Updated tool descriptions in `packages/coding/src/tools/registry.ts`
   - Updated `packages/coding/src/prompts/system.ts` to steer the model toward `grep` + segmented `read`, and toward narrower follow-up calls after truncation hints

### Validation

- `npm run build:packages`
- `npx tsc -p tsconfig.json --noEmit`
- `npx vitest run packages/coding/src/tools/bash.test.ts packages/coding/src/tools/truncate.test.ts packages/coding/src/tools/tool-result-policy.test.ts packages/coding/src/tools/read.test.ts packages/coding/src/tools/grep.test.ts tests/kodax_core.test.ts tests/interactive.test.ts`

### Notes

- Compaction still exists as a downstream safety mechanism, but it is no longer the only line of defense against oversized tool output.
- Spill files are written to the user-level KodaX tool-results directory by default, not into the project workspace.
