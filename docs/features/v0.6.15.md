## FEATURE_033: REPL Parallel Toggle (/parallel)

**Status**: Planned
**Priority**: Medium
**Category**: Enhancement

### 1. 概述

在 REPL 交互模式中添加 `/parallel` 命令，允许用户在运行时动态切换并行工具执行模式，无需重启 KodaX。

**核心目标**:
1. **运行时切换** — `/parallel on` / `/parallel off` 动态切换，无需重启
2. **状态可见** — `/parallel` 显示当前状态，状态栏显示 parallel 指示
3. **持久化** — 选项保存到会话配置，下次启动时保持

**背景**:
- 当前 `--parallel` / `-j` 只能在 CLI 启动时设置
- REPL 中无法查看或切换 parallel 状态
- `KodaXOptions.parallel` 通过 `currentOptionsRef` 传递，已有运行时可变性基础

### 2. 影响范围

**需要修改的文件**:
- `packages/repl/src/interactive/commands.ts` — 注册 `/parallel` 命令
- `packages/repl/src/ui/components/status-bar.tsx` — 显示 parallel 状态指示（可选）

**依赖**:
- `KodaXOptions.parallel` — 已存在的选项字段
- `currentOptionsRef` — 已存在的响应式引用，支持运行时更新

### 3. 技术方案

#### 3.1 命令定义

```typescript
// /parallel 命令行为
{
  name: 'parallel',
  aliases: ['pm'],
  description: 'Toggle parallel tool execution mode',

  // /parallel        → 显示当前状态
  // /parallel on     → 启用并行
  // /parallel off    → 禁用并行
  // /parallel toggle → 切换状态
}
```

#### 3.2 实现逻辑

```typescript
async function handleParallel(args: string): CommandResult {
  const current = currentOptionsRef.current.parallel;

  switch (args.trim().toLowerCase()) {
    case '':
      // 显示当前状态
      return statusMessage(`Parallel: ${current ? 'ON' : 'OFF'}`);
    case 'on':
      updateOptions({ parallel: true });
      return statusMessage('Parallel mode enabled');
    case 'off':
      updateOptions({ parallel: false });
      return statusMessage('Parallel mode disabled');
    case 'toggle':
      updateOptions({ parallel: !current });
      return statusMessage(`Parallel: ${!current ? 'ON' : 'OFF'}`);
    default:
      return errorMessage('Usage: /parallel [on|off|toggle]');
  }
}
```

### 4. 接口契约

```
/parallel              # 显示当前 parallel 状态
/parallel on           # 启用并行工具执行
/parallel off          # 禁用并行工具执行
/parallel toggle       # 切换并行状态
```

### 5. 实现步骤

1. **注册命令** — 在 `commands.ts` 中添加 `/parallel` 命令处理
2. **状态读取** — 从 `currentOptionsRef.current.parallel` 读取当前状态
3. **状态更新** — 通过 `updateOptions({ parallel })` 更新运行时状态
4. **帮助集成** — 添加到 `/help` 命令输出

### 6. 验收标准

- [ ] `/parallel` 命令在 REPL 中可用
- [ ] `/parallel on` 启用并行执行
- [ ] `/parallel off` 禁用并行执行
- [ ] `/parallel toggle` 正确切换状态
- [ ] 无参数时显示当前状态
- [ ] 状态变更在后续 tool 调用中生效

## FEATURE_036: DeepSeek 内置 Provider 支持

**Status**: Planned
**Priority**: Medium
**Category**: Enhancement

### 1. 需求概述

为 KodaX 添加 DeepSeek 作为第 11 个内置 provider，基于 DeepSeek-V3.2。

**核心目标**:
1. **内置 Provider** — 注册为 `deepseek` 内置 provider，无需用户手动配置 custom provider
2. **双模型支持** — `deepseek-chat`（默认，128K 上下文）和 `deepseek-reasoner`（推理模式，64K 输出）
3. **原生推理** — `deepseek-reasoner` 通过 `native-effort` reasoning capability 支持 CoT 推理
4. **完整功能** — 支持 JSON Output、Tool Calls、FIM Completion

**DeepSeek V3.2 API 规格**:
- Base URL: `https://api.deepseek.com`
- API Key 环境变量: `DEEPSEEK_API_KEY`
- 协议: OpenAI 兼容
- 上下文窗口: 128K
- `deepseek-chat`: 非思考模式，默认 4K / 最大 8K 输出
- `deepseek-reasoner`: 思考模式（CoT），默认 32K / 最大 64K 输出
- 支持: JSON Output, Tool Calls, FIM Completion（仅 deepseek-chat）
